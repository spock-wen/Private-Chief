# SpockChef UI/UX 视觉预览与需求优化方案

根据您的要求，我将结合 [PROJECT_STANDARDS.md](file:///d:/Devlopment/LiteProgram/Private%20chef/PROJECT_STANDARDS.md) 的视觉规范，为您规划并生成一套高保真 UI 预览。

## 1. 视觉设计语言 (Design Language)
*   **核心色调**：
    *   `Primary`: `#D97706` (琥珀金) - 用于主按钮、核心状态图标。
    *   `Background`: `#FFFBF0` (暖米白) - 营造温馨的家庭氛围。
    *   `Text`: `#431407` (深褐) - 提升阅读质感，避免纯黑的冷硬感。
*   **设计风格**：
    *   **大圆角 (16px)**：所有卡片和容器均采用柔和圆角。
    *   **弥散阴影**：使用暖色调阴影，增加页面的深度与呼吸感。
    *   **排版**：采用富有叙事感的标题字体，强调“私厨”的艺术性。

## 2. 核心页面预览规划 (UI Previews)
我将为您构建以下三个关键场景的交互预览：

### A. 主人筹备页 (Host Dashboard - Planning)
*   **功能**：管理候选菜单、一键开启投票。
*   **设计亮点**：优雅的瀑布流菜品卡片，带有点选动画；明显的“状态指示灯”。

### B. 客人投票互动页 (Guest Interaction - Voting)
*   **功能**：查看主人预选菜品、点赞投票。
*   **设计亮点**：极简的“想吃”按钮，带有心跳动效；实时显示的“大家最想吃”热度排行。

### C. 账单结算弹窗 (Bill Settlement - Archived)
*   **功能**：录入总价，自动计算 AA。
*   **设计亮点**：清晰的金额输入框，温馨的“人均分摊”提示卡片。

## 3. 需求优化项 (PM Insights)
在生成预览的同时，我已识别并将在设计中体现以下优化：
1.  **身份重连**：在客人端增加“这是我之前的选择”自动关联提示。
2.  **口味备注**：加入饭桌时可选“忌口/偏好”标签。
3.  **状态锁定**：在 UI 上清晰展示“投票已锁定，菜单不可修改”的提示。
